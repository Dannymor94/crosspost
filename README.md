# crosspost

Локальный кросс-постинг: загрузил контент → выбрал каналы → опубликовалось во все разом → видишь поканальный результат. Для себя и небольшого круга (3–5 человек). Ориентир — каналы, работающие в РФ; без зависимости от заблокированных облаков.

Контент **готовый, на входе.** Система доставляет, а не генерирует.

## Каналы

**API-тир** (надёжный): Telegram (Telethon), ВКонтакте, YouTube, Telegraph.
**Браузерный тир** (хрупкий, Playwright): WhatsApp, Instagram, Дзен, Яндекс Бизнес.

Граница API↔браузер — главный архитектурный принцип; от неё зависят релогин, IP, прокси и масштабирование. Подробно — в `PROJECT_GUIDE.md`.

## Старт (планируется)

**Быстрый путь к MVP** (CLI постит в Telegram + ВК, без веба/очереди/БД):
```bash
make install-mvp                  # только ядро + dev, ставится быстро
make test                         # тесты ядра
cp .env.example runtime/.env      # заполнить 5 ключей (см. SETUP.md)
make smoke                        # реальная отправка в TG+ВК
```
Подробный порядок — в `MILESTONES.md` и `PLAN.md`. Инфраструктура (Redis/Postgres/веб) — post-MVP.

## Безопасность (важно)

Проект хранит токены и сессии (при друзьях — чужие аккаунты). Перед первым коммитом проверить, что `git status` не показывает `.env`, `runtime/`, `*.session`, `vault*`. Репозиторий — приватный. Детали — в `PROJECT_STRUCTURE.md`.
